# Contributing to enagramJS

Thank you for your interest in contributing! This project welcomes improvements, bug fixes, and new content.

## How to Contribute
1. Fork the repository and clone your fork
2. Create a new branch for your feature or fix
3. Make your changes (see guidelines below)
4. Add or update tests in `tests/` if relevant
5. Update or add documentation in `docs/` if needed
6. Commit and push your branch
7. Open a Pull Request with a clear description

## Guidelines
- Keep code modular and well-documented
- Use descriptive commit messages
- Prefer pure functions in `utils/` and reusable components in `components/`
- Add tests for new features or bug fixes
- Update documentation for any user-facing or architectural changes

## Code Style
- Use modern ES6+ JavaScript
- Prefer functional React components
- Use consistent formatting (Prettier or VS Code default)

## Reporting Issues
- Use GitHub Issues for bugs, feature requests, or questions
- Provide clear steps to reproduce bugs
